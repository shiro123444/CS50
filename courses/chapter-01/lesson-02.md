# Scratch 编程基础

## 什么是 Scratch？

Scratch 是由 MIT 媒体实验室开发的可视化编程语言，专为初学者设计。通过拖拽积木块的方式，你可以创建动画、游戏和交互式故事，而无需编写复杂的代码。

![Scratch Interface](../../resources/images/scratch-interface.png)

## Scratch 界面介绍

Scratch 的界面主要分为四个区域：

1. **舞台区**（Stage）- 程序运行的地方
2. **精灵列表**（Sprite List）- 管理所有角色
3. **积木区**（Blocks Palette）- 各种编程积木
4. **脚本区**（Scripts Area）- 组装积木的地方

## 核心编程概念

### 1. 事件（Events）

程序需要一个起点，在 Scratch 中通过事件积木来触发：

```
当绿旗被点击
当按下空格键
当这个精灵被点击
```

### 2. 动作（Motion）

控制精灵的移动：

- 移动 10 步
- 转向 90 度
- 移到 x: 0 y: 0
- 滑行到指定位置

### 3. 外观（Looks）

改变精灵的外观：

- 说 "Hello!" 2 秒
- 切换造型
- 显示/隐藏
- 改变大小

### 4. 声音（Sound）

播放音效和音乐：

- 播放声音
- 播放音符
- 改变音量

## 控制结构

### 条件判断（If-Then）

```
如果 <条件> 那么
  执行这些积木
```

示例：如果碰到边缘，就反弹

### 循环（Loops）

**重复执行：**

```
重复 10 次
  移动 10 步
```

**永远重复：**

```
重复执行
  移动 10 步
  如果碰到边缘就反弹
```

**重复直到：**

```
重复直到 <碰到颜色>
  移动 10 步
```

## 变量和数据

### 变量（Variables）

变量用来存储数据：

```
创建变量 "分数"
将 分数 设为 0
将 分数 增加 1
```

### 列表（Lists）

列表可以存储多个值：

```
创建列表 "名字"
将 "Alice" 加入 名字
```

## 实践示例：制作一个简单游戏

让我们创建一个"接球游戏"：

### 游戏设计

| 元素 | 说明 |
|------|------|
| 玩家 | 用鼠标控制的接球板 |
| 球 | 从上方落下的物体 |
| 分数 | 记录接到的球数 |
| 游戏结束 | 球落地时游戏结束 |

### 接球板脚本

```
当绿旗被点击
重复执行
  将 x 坐标设为 鼠标的 x 坐标
```

### 球的脚本

```
当绿旗被点击
将 分数 设为 0
重复执行
  移到 x: (随机数 -200 到 200) y: 180
  重复直到 <y 坐标 < -180>
    将 y 坐标增加 -5
    如果 <碰到 接球板> 那么
      将 分数 增加 1
      播放声音 pop
      移到 x: (随机数 -200 到 200) y: 180
  说 "游戏结束！" 2 秒
  停止 全部
```

## 计算思维概念

通过 Scratch，我们学习了重要的计算思维概念：

### 算法（Algorithms）

> 算法是解决问题的步骤序列

示例：制作三明治的算法
1. 拿出两片面包
2. 在一片面包上涂花生酱
3. 在另一片面包上涂果酱
4. 将两片面包合在一起

### 抽象（Abstraction）

隐藏复杂的细节，只关注重要的部分。在 Scratch 中，我们使用"自定义积木"来实现抽象。

### 分解（Decomposition）

将大问题分解成小问题。例如，制作游戏时：
- 先实现玩家控制
- 再实现敌人移动
- 最后添加计分系统

### 模式识别（Pattern Recognition）

识别重复的模式，使用循环来简化代码。

## 常见编程模式

### 1. 跟随鼠标

```
重复执行
  移到 鼠标指针
```

### 2. 键盘控制

```
重复执行
  如果 <按下 右箭头 键> 那么
    将 x 坐标增加 10
  如果 <按下 左箭头 键> 那么
    将 x 坐标增加 -10
```

### 3. 碰撞检测

```
如果 <碰到 敌人> 那么
  说 "哎呀！" 2 秒
  移到 x: 0 y: 0
```

### 4. 随机移动

```
重复执行
  面向 (随机数 0 到 360) 度
  移动 10 步
  如果碰到边缘就反弹
```

## Week 0 作业提示

创建你的 Scratch 项目时，考虑以下要求：

- [ ] 至少使用两个精灵
- [ ] 至少使用三个脚本
- [ ] 使用至少一个条件判断
- [ ] 使用至少一个循环
- [ ] 使用至少一个变量
- [ ] 让项目具有交互性

### 项目创意

- 🎮 简单的游戏（迷宫、接球、打砖块）
- 🎨 交互式动画或故事
- 🎵 音乐创作工具
- 🧮 数学小工具（计算器、测验）

## 调试技巧

当程序不按预期工作时：

1. **使用"说"积木**
   - 显示变量的值
   - 确认代码是否执行

2. **逐步测试**
   - 一次添加一个功能
   - 确保每个部分都能工作

3. **检查常见错误**
   - 事件积木是否正确
   - 循环是否会结束
   - 变量是否正确初始化
