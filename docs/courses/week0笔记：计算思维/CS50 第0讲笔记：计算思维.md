# 二进制系统深度解析
* 日常生活中，我们使用的计数方式是一进制
* 而计算机使用的则是二进制（0和1）
## 二进制基础概念

- **位（Bit）**：计算机中最小的数据单位，值为0或1
- **字节（Byte）**：8位组成1个字节，可表示256种状态
- **字（Word）**：计算机处理数据的单位长度

##  二进制转换方法

**十进制转二进制**：

- 除2取余法：不断除以2，记录余数，最后倒序排列
- 示例：13的二进制转换
    
    ```
    13 ÷ 2 = 6 余 1
    6 ÷ 2 = 3 余 0
    3 ÷ 2 = 1 余 1
    1 ÷ 2 = 0 余 1
    结果：1101
    ```
    

**二进制转十进制**：

- 按权展开求和法
- 示例：1101的十进制计算
    
    ```
    1×2³ + 1×2² + 0×2¹ + 1×2⁰
    = 8 + 4 + 0 + 1 = 13
    ```
# 其他进制转换方法
## 1. 八进制（Octal）系统

### 1.1 八进制基础概念

- **基数**：8
- **数字范围**：0-7
- **表示方法**：在数字前加`0`前缀（如`075`）

### 1.2 八进制转换方法

**十进制转八进制**：

- 除8取余法
- 示例：83的八进制转换
    
    ```
    83 ÷ 8 = 10 余 3
    10 ÷ 8 = 1 余 2
    1 ÷ 8 = 0 余 1
    结果：0123（八进制）
    ```
    

**八进制转十进制**：

- 按权展开求和法
- 示例：0123的十进制计算
    
    ```
    1×8² + 2×8¹ + 3×8⁰
    = 64 + 16 + 3 = 83
    ```
    

**八进制与二进制转换**：

- 1位八进制 = 3位二进制
- 示例：`5₈` = `101₂`
- 快速转换：`0123₈` = `001 010 011₂`

## 2. 十六进制（Hexadecimal）系统

### 2.1 十六进制基础概念

- **基数**：16
- **数字范围**：0-9, A-F（A=10, B=11, C=12, D=13, E=14, F=15）
- **表示方法**：在数字前加`0x`前缀（如`0x1A3`）

### 2.2 十六进制转换方法

**十进制转十六进制**：

- 除16取余法
- 示例：419的十六进制转换
    
    ```
    419 ÷ 16 = 26 余 3
    26 ÷ 16 = 1 余 10（A）
    1 ÷ 16 = 0 余 1
    结果：0x1A3
    ```
    

**十六进制转十进制**：

- 按权展开求和法
- 示例：0x1A3的十进制计算
    
    ```
    1×16² + 10×16¹ + 3×16⁰
    = 256 + 160 + 3 = 419
    ```
    

**十六进制与二进制转换**：

- 1位十六进制 = 4位二进制
- 示例：`A₍₁₆₎` = `1010₂`
- 快速转换：`0x1A3₍₁₆₎` = `0001 1010 0011₂`

## 3. 各进制系统对比

|进制|基数|数字范围|前缀|主要应用|
|---|---|---|---|---|
|二进制|2|0-1|无|计算机内部处理|

|十六进制|16|0-9, A-F|0x|内存地址、颜色编码|

## 5. 转换技巧总结

### 5.1 快速转换方法

**二进制 ↔ 八进制**：

- 从右向左，每3位一组
- 不足3位左侧补零
- 直接转换每组为八进制

**二进制 ↔ 十六进制**：

- 从右向左，每4位一组
- 不足4位左侧补零
- 直接转换每组为十六进制

### UTF-8编码(Unicode Transformation Format)
99%的前端写网页时都会加上，99%的后端工程师新建数据库表时都会加上DEFAULT CHARSET=utf8（剩下的1%应该是忘了写）。
Unicode赋予了全世界所有文字和符号一个独一无二的数字编号，UTF8所做的事情就是把这个数字编号表示出来。
**特点：​**
- 可变长度编码（1-4字节）
- 兼容ASCII
- 互联网最常用的Unicode编码

| ------------------ | ------------------------------------- | --- | --------------- | --- | --- |
| U+0000 ~ U+007F    | `0xxxxxxx`                            | 1   | `'A'`= U+0041   |     |     |

> 💡 规则说明：
> 
> - 第一字节以 `0` 开头 → 1 字节；
> - 第一字节以 `110` 开头 → 2 字节；
> - 第一字节以 `1110` 开头 → 3 字节；
> - 第一字节以 `11110` 开头 → 4 字节；
> - 后续字节均以 `10` 开头。

这种巧妙设计，把Unicode的数值和每个字的字节数融合在一起，最坏情况是6个字节表示一个字，已经足够表示世界上所有语言的所有文字了。

- 属于 1 字节范围 → 直接输出：`01000001` → 十六进制：`0x41`

##### 示例2：带重音字母 `'é'`（U+00E9）

- 二进制：`11101001`
- 属于 2 字节范围（U+0080 ~ U+07FF）
- 拆分：高5位 `11101` + 低6位 `001001`
- 编码：
    - 第一字节：`110` + `11101` → `11011101`
    - 第二字节：`10` + `001001` → `10001001`
- 十六进制：`0xCD 0x89`

✅ 结果：`CD 89`（2 字节）

---

##### 示例3：汉字 `'中'`（U+4E2D）

- 二进制：`01001110 00101101`
- 属于 3 字节范围（U+0800 ~ U+FFFF）
- 拆分：高4位 `0100` + 中6位 `111000` + 低6位 `101101`
- 编码：
    - 第一字节：`1110` + `0100` → `11100100`
    - 第二字节：`10` + `111000` → `10111000`
    - 第三字节：`10` + `101101` → `10101101`
- 十六进制：`0xE4 0xB8 0xAD`

✅ 结果：`E4 B8 AD`（3 字节）

---

##### 示例4：表情符号 `'😀'`（U+1F600）

- 二进制：`00011111 01100000 00000000`
- 属于 4 字节范围（U+10000 ~ U+10FFFF）
- 拆分：高3位 `000` + 中6位 `111110` + 中6位 `110000` + 低6位 `000000`
- 编码：
    - 第一字节：`11110` + `000` → `11110000`
    - 第二字节：`10` + `111110` → `10111110`
    - 第三字节：`10` + `110000` → `10110000`
    - 第四字节：`10` + `000000` → `10000000`
- 十六进制：`0xF0 0x9F 0x98 0x80`

✅ 结果：`F0 9F 98 80`（4 字节）

**优点：​**

- 英文文本效率高（与ASCII相同）
- 无字节序问题
- 容错能力强

# 算法（Algorithms）
###  算法基本特性

- **有穷性**：算法必须在有限步骤内结束
- **确定性**：每个步骤都有明确含义
- **可行性**：每个步骤都可以执行
- **输入**：零个或多个输入
- **输出**：一个或多个输出
### 搜索算法详细对比

**顺序搜索算法**：

```伪代码
函数 顺序搜索(列表, 目标值):
    对于 列表中的每个元素:
        如果 当前元素 == 目标值:
            返回 元素位置
    返回 未找到
```

- 优点：实现简单，无需排序
- 缺点：效率低，大数据集性能差

**这个时候就要请出cs50每年第一次讲算法都会讲到的“分治法”（也叫二分查找）**
**二分搜索算法**：

```伪代码
函数 二分搜索(有序列表, 目标值):
    左边界 = 0
    右边界 = 列表长度 - 1
    
    当 左边界 <= 右边界:
        中间位置 = (左边界 + 右边界) // 2
        如果 列表[中间位置] == 目标值:
            返回 中间位置
        否则如果 列表[中间位置] < 目标值:
            左边界 = 中间位置 + 1
        否则:
            右边界 = 中间位置 - 1
    返回 未找到
```

- 前提条件：数据必须有序
- 优点：搜索效率高!

# 伪代码（Pseudocode）编写详解

## 1. 伪代码基本概念

### 1.1 什么是伪代码？
- **定义**：用自然语言和简单符号描述算法逻辑的"假代码"
- **特点**：忽略编程语言的具体语法，专注于算法思想
- **目的**：在编写实际代码前理清思路，便于沟通和设计

### 1.2 伪代码的优势
- **语言无关**：不依赖特定编程语言
- **易于理解**：使用自然语言描述
- **快速原型**：快速验证算法可行性
- **团队协作**：便于非技术人员理解

## 2. 伪代码基本结构

### 2.1 变量声明
```
设置 变量名 = 初始值
```
**示例**：
```
设置 学生数量 = 0
设置 总分 = 0
设置 平均分 = 0
```

### 2.2 输入输出语句
```
输入：变量名
输出：变量名 或 文本信息
```
**示例**：
```
输入：学生成绩
输出：平均分
输出："计算完成"
```

## 3. 控制结构详细语法

### 3.1 条件语句
**基本格式**：
```
如果 条件:
    执行操作1
否则:
    执行操作2
```

**多条件判断**：
```
如果 条件1:
    操作1
否则如果 条件2:
    操作2
否则如果 条件3:
    操作3
否则:
    默认操作
```

**嵌套条件**：
```
如果 外层条件:
    如果 内层条件:
        操作1
    否则:
        操作2
否则:
    操作3
```

### 3.2 循环结构
**当循环（While）**：
```
当 条件成立:
    重复执行操作
    更新循环条件
```

**对于循环（For）**：
```
对于 i 从 起始值 到 结束值:
    执行操作
```

**遍历集合**：
```
对于 每个元素 in 集合:
    处理元素
```

## 5. 伪代码编写规范

### 5.1 命名规范

- 使用统一的缩进（通常4个空格）
- 相关的代码块要对齐
- 空行分隔不同的逻辑部分

```

```

## 6. 完整示例分析

### 6.1 计算平均分算法
```
函数 计算平均分(成绩列表):
    // 初始化变量
    设置 总分 = 0
    设置 学生数量 = 成绩列表.长度
    
    // 计算总分
    对于 每个成绩 in 成绩列表:
        总分 = 总分 + 成绩
    
    // 计算平均分
    如果 学生数量 > 0:
        平均分 = 总分 / 学生数量
        返回 平均分
    否则:
        返回 0
```

### 6.2 查找最大值算法
```
函数 查找最大值(数字列表):
    // 假设第一个元素是最大值
    设置 最大值 = 数字列表[0]
    
    // 遍历比较
    对于 i 从 1 到 数字列表.长度-1:
        如果 数字列表[i] > 最大值:
            最大值 = 数字列表[i]
    
    返回 最大值
```

### 6.3 二分搜索算法
```
函数 二分搜索(有序列表, 目标值):
    // 初始化边界
    设置 左边界 = 0
    设置 右边界 = 有序列表.长度 - 1
    
    // 搜索循环
    当 左边界 <= 右边界:
        设置 中间位置 = (左边界 + 右边界) // 2
        
        如果 有序列表[中间位置] == 目标值:
            返回 中间位置  // 找到目标
        否则如果 有序列表[中间位置] < 目标值:
            左边界 = 中间位置 + 1  // 搜索右半部分
        否则:
            右边界 = 中间位置 - 1  // 搜索左半部分
    
    返回 -1  // 未找到
```

## 7. 常见算法模式

### 7.1 累加模式
```
设置 总和 = 0
对于 每个元素 in 列表:
    总和 = 总和 + 元素
```

### 7.2 计数模式
```
设置 计数 = 0
对于 每个元素 in 列表:
    如果 元素满足条件:
        计数 = 计数 + 1
```

### 7.3 查找模式
```
设置 找到 = 假
对于 每个元素 in 列表:
    如果 元素 == 目标值:
        找到 = 真
        跳出循环
```

### 7.4 极值模式
